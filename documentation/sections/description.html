3. Evaluation Result ontology Description back to ToC

Figure 1 shows an overview of the Evaluation Result ontology and how the concepts relate to one another.

The Evaluation class represents the process of evaluation, in which a particular product, service or action is evaluated (EvaluationSubject class). Each evaluation produces a quality value (represented by the QualityValue class) which is related to a particular quality measure (qmo:QualityMeasure), and uses some input data (e.g, a document) which is represented by the EvaluationData class. Finally, each evaluation subject belongs to a certain category of products, services or actions, which is represented by the SubjectCategory class.

Figure 1. A high level overview of the Evaluation Result ontology.

The Evaluation Result ontology reuses the Ontology of units of Measure and related concepts (om) [1] and the W3C's Time Ontology. Each evaluation is performed in a specific moment in time represented by the time:Instant class, and each quality value is expressed in a unit of measure (om:Unit_of_measure) and belongs to a particular scale (om:Measurement_scale), as defined in the Quality Model ontology.

The Evaluation Result ontology has been evaluated using the OOPS! pitfall scanner [2].

3.1 Illustrative Example

Listing 1 presents a short illustrative example of evaluation results data related to one ontology matching tool (YAM++), which continues from the example presented in the Quality Model ontology. For the sake of simplicity, datatype declarations are omitted.

@prefix ex: <http://example.com> .
@prefix qmo: <http://purl.org/net/QualityModel#> .
@prefix eval: <http://purl.org/net/EvaluationResult#> .
@prefix time: <http://www.w3.org/2006/time#> .
@prefix dc: <http://purl.org/dc/terms/> .

<ex:Evaluation1> a eval:Evaluation ;
eval:evaluatedSubject <ex:YAM++> ;
eval:inputData <ex:Ontology1> ;
eval:inputData <ex:Ontology2> ;
eval:inputData <ex:ExpectedAlignment> ;
eval:producedQualityValue <ex:QualityValue1> ;
eval:time <ex:Instant1> .

<ex:YAM++> a eval:EvaluationSubject ;
dc:title "YAM++" ;
eval:belongsToCategory <ex:OntologyMatchingTools> .

<ex:QualityValue1> a eval:QualityValue ;
eval:hasLiteralValue "0.86" ;
eval:obtainedFrom <ex:Evaluation1> ;
eval:forMeasure <ex:AverageAlignmentPrecision> ;
eval:isMeasuredOnScale <ex:PrecisionScale> .

<ex:Instant1> a time:Instant ;
time:inXSDDateTime "2015-03-13T03:25:00" .

<ex:Ontology1> a eval:EvaluationData .
<ex:Ontology2> a eval:EvaluationData .
<ex:ExpectedAlignment> a eval:EvaluationData .

Listing 1. Illustrative example of evaluation result data related to YAM++ ontology matching tool.
